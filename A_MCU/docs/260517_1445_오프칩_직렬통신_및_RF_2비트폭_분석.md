## 1. 칩 외부 직렬 통신(Serial Communication) 구조 및 동작 원리

본 MCU 설계([my_mcu_top.v](file:///home/hosung/Ho/MPW/MPW_workspace/Bit_Up/A_MCU/src/rtl/my_mcu_top.v))는 SOP-28 패키지와 같이 핀 수가 극도로 제한된 타겟을 고려하여, **레지스터 파일(RF)과 명령어/데이터 메모리(Instruction/Data Memory) 모두를 칩 외부(예: 외부 Pico 보드)에 두고 4핀 SPI 유사 직렬 인터페이스를 통해 접근**하도록 커스텀되어 있습니다.

칩 외부로 나가는 직렬 통신 채널은 크게 **두 가지 독립적인 인터페이스**로 나뉩니다.

### A. 외부 RF 브리지 (External Register File Bridge)
- **연결 목적**: SERV CPU 코어의 레지스터 파일(RF) 및 CSR 공간을 오프칩 메모리에 시리얼하게 저장하고 읽어오기 위함.
- **포트 정의**:
  - `o_rf_sync` (Sync/Active-High CS): 전송 트랜잭션의 시작과 유효 기간을 지시하는 동기화 신호.
  - `o_rf_sck` (Serial Clock): 데이터 쉬프트를 위한 전송 클럭. 외부 Pico가 데이터를 안정적으로 캡처하도록 고속 클럭(`i_clk_fast`)의 반전 위상(`~i_clk_fast`)으로 동작함.
  - `o_rf_mosi` (Master Output): MCU에서 외부 RF로 주소, 제어 신호, 쓰기 데이터를 전송하는 직렬 출력 핀.
  - `i_rf_miso` (Master Input): 외부 RF로부터 데이터를 읽어들이는 직렬 입력 핀.
- **프레임 포맷 (총 `raw + 8` 비트 = CSR 4개 기준 14비트)**:
  - `req_key = {i_wen, i_ren, target_reg, target_bit, i_wdata}`
  - `{Write Enable(1비트), Read Enable(1비트), 레지스터 번호(6비트), 비트 오프셋(4비트), 쓰기 데이터(2비트)}`로 구성됩니다.
  - 이 14비트 데이터가 `o_rf_mosi`를 통해 순차적으로 쉬프트되어 외부 RF로 전송되며, **읽기 트랜잭션 시에는 프레임 전송의 가장 마지막 2클럭 동안 `i_rf_miso`를 통해 2비트의 데이터를 읽어옵니다.**

### B. 외부 메모리 브리지 (External Instruction/Data Memory Bridge)
- **연결 목적**: CPU의 명령어 페치(Instruction Fetch) 및 데이터 메모리 접근(Load/Store)을 칩 외부의 공유 메모리 장치와 통신하여 수행하기 위함.
- **포트 정의**:
  - `o_mem_sync` / `o_mem_sck` / `o_mem_mosi` / `i_mem_miso` (동일하게 4핀 구조 사용)
- **프레임 포맷 (총 70비트, MSB First)**:
  - `{WE(1비트), IBUS access flag(1비트), Byte Select(4비트), 32비트 물리 주소(32비트), 32비트 쓰기 데이터 또는 Dummy(32비트)}`
- **읽기/쓰기 동작**:
  - **읽기(Read)**: 70비트 프레임 전송 중 마지막 32비트 구간(쓰기 데이터/더미 송신 구간) 동안 외부 메모리가 `i_mem_miso`를 통해 32비트 데이터를 시리얼하게 송신하며, 브리지가 이를 `shift_rx` 버퍼에 누적한 후 CPU로 Wishbone ACK와 함께 반환합니다.
  - **쓰기(Write)**: 70비트 프레임을 모두 내보낸 후 외부 장치의 응답 없이 바로 내부 Wishbone ACK를 발생시켜 고속으로 트랜잭션을 마칩니다.

## 2. `serv_rf_ram_if`의 2비트 전송 폭 (`width=2`)의 의미

`serv_rf_ram_if.v`에 설정되어 있는 `parameter width=2` (전송 폭 2비트)는 **비트-시리얼 SERV 코어와 2비트 외부 직렬 RF 스토리지 인터페이스 간의 데이터 변환 규격**을 의미합니다.

### A. 비트-시리얼(Bit-serial)과 외부 인터페이스 간 완충
- SERV 코어 자체는 매 사이클마다 레지스터 2개(rs1, rs2)에서 1비트씩 읽고, 1비트씩 쓰는 **비트-시리얼(W=1)** 코어입니다.
- 하지만 오프칩 통신을 매 비트마다 1비트씩 트랜잭션을 발생시켜 칩 외부로 처리하면 직렬 전송의 프로토콜 오버헤드가 극도로 심해집니다.
- 이를 개선하기 위해 **외부 RF 통신 인터페이스를 2비트 단위(`width=2`, `o_wdata` 및 `i_rdata`가 2비트 폭)**로 설계하여, 프로토콜 오버헤드를 낮추면서도 전송 핀 개수 및 시리얼 동작의 효율성을 극대화하도록 절충한 것입니다.

### B. 읽기 및 쓰기에서의 2비트 처리 과정

#### 1) 쓰기 동작 (Write)
- SERV 코어가 `i_wdata0`과 `i_wdata1` 포트를 통해 각각 1비트씩(W=1)을 내보내면, RF 인터페이스가 이를 내부 쉬프트 레지스터(`wdata0_r`, `wdata1_r`)에 모았다가 `width=2` 폭에 맞추어 2비트씩 외부 `serv_rf_ram` 및 칩 외부로 출력(`o_wdata`)합니다.
- 이 과정은 원래 SERV RAM 어댑터 타이밍을 준수하여 동작합니다.

#### 2) 읽기 동작 (Read & Prefetch)
- SERV 코어가 레지스터 데이터를 읽기 시작하기 전에, RF 인터페이스 모듈이 **미리(Prefetch)** 외부 브리지에 접근하여 필요한 레지스터의 데이터를 읽어옵니다.
- 이때 외부로부터 한 번에 2비트 폭(`i_rdata` [1:0])으로 시리얼 데이터를 수신합니다.
- 수신된 데이터는 32비트 크기의 프리페치 버퍼(`read_buf0`, `read_buf1`)에 채워진 뒤, SERV 코어가 활성화되면 매 사이클 1비트씩(`o_rdata0`, `o_rdata1`) 비트-시리얼하게 풀어내어 SERV CPU 코어에 최종 공급합니다.

### C. 요약
즉, `serv_rf_ram_if`의 **2비트 전송 폭**은
> **"비트-시리얼(W=1)로 동작하는 SERV CPU 코어와 외부 오프칩 2비트 시리얼 스토리지 규격 간의 데이터 폭을 매칭시켜 주고, 프리페치(Prefetch) 및 버퍼링을 통해 시리얼 전송 효율을 향상시키기 위한 인터페이스 데이터 폭 규격"**

을 의미합니다.
