# 1mm x 1mm 면적 충족을 위해 포기한 기능 정리

작성 시각: 260520_1444

## 목적

ETRI 0.5um MPW 제출 조건인 `1mm x 1mm` 안에 `my_mcu_top`을 넣기 위해 적용한 RTL 및 QFlow physical profile 변경을 정리한다.

이 문서는 단순 최적화 기록이 아니라, 면적을 맞추기 위해 어떤 기능, 정확성 보장, 검증 범위, 물리 구현 여유를 포기했는지를 빠짐없이 남기는 용도다.

## 최종 결과

현재 physical 전용 profile은 placement 및 route 기준으로 `1mm x 1mm` 조건을 충족했다.

### Logical synthesis 면적

`backend_physical_qflow/source/my_mcu_top_custom.ys` 기준 Yosys liberty area:

```text
Chip area for module '\my_mcu_top': 501345.000000
DFFPOSX1: 264
Total cells: 1408
```

### QFlow placement 면적

`backend_physical_qflow/log/place.log`:

```text
Number of cells = 1476, total width = 2568600
Total cell width   :2.57e+06
Total cell height  :5.31e+06
Total cell area    :9.25e+09
Total core area    :9.25e+09
Initial core layout: (150 150) to (95850 97350) (scale um * 100)
```

해석:

- core layout width: `(95850 - 150) / 100 = 957.0um`
- core layout height: `(97350 - 150) / 100 = 972.0um`
- core layout: 약 `0.957mm x 0.972mm`

### DEF die area

`backend_physical_qflow/layout/my_mcu_top.def`:

```text
DIEAREA ( -600 0 ) ( 96600 98100 ) ;
```

해석:

- DEF width: `(96600 - (-600)) / 100 = 972.0um`
- DEF height: `(98100 - 0) / 100 = 981.0um`
- DEF die area: 약 `0.972mm x 0.981mm`

### Route 결과

`backend_physical_qflow/log/route.log`:

```text
Final: No failed routes!
```

즉, 현재 density `1.00` 배치에서도 qrouter 기준 미배선 net은 없다.

## 중요한 전제

현재 1mm 조건을 만족한 것은 `canonical default RTL` 전체 기능이 아니다.

기본 RTL은 기존 기능 검증을 위해 다음과 같은 안전한 기본값을 유지한다.

- `RV32E_MODE = 0`
- `USE_RF_INTERLOCK = 1`
- `DISABLE_DBUS = 0`
- `SINGLE_RF_READ = 0`
- `RESET_STRATEGY = "MINI"`
- `USE_EXT_SYS_CLK = 0`

반면, 면적을 맞춘 physical 전용 profile은 `backend_physical_qflow/source/my_mcu_top_custom.ys`에서 다음처럼 강제된다.

```tcl
chparam -set USE_EXT_SYS_CLK 1 my_mcu_top
chparam -set RV32E_MODE 1 my_mcu_top
chparam -set MEM_ADDR_BITS 11 my_mcu_top
chparam -set USE_RF_INTERLOCK 0 my_mcu_top
chparam -set DISABLE_DBUS 1 my_mcu_top
chparam -set SERV_PRE_REGISTER 1 my_mcu_top
chparam -set SINGLE_RF_READ 1 my_mcu_top
chparam -set RESET_STRATEGY "NONE" my_mcu_top
```

따라서 tape-out 후보는 일반 RV32I MCU가 아니라, 매우 제한된 `area-first proof-of-silicon core`에 가깝다.

## 포기한 기능 및 보장

### 1. RV32I 전체 레지스터 파일 포기

적용:

```verilog
parameter integer RV32E_MODE = 1
```

변경 내용:

- GPR 수를 x0-x31에서 x0-x15로 축소했다.
- RF adapter의 storage depth도 32개 GPR 기준에서 16개 GPR 기준으로 축소했다.

포기한 것:

- RV32I full integer register file.
- x16-x31 사용.
- 일반 RV32I ABI 호환성.
- 기존 RV32I 컴파일러가 자유롭게 생성하는 register allocation 결과.

남는 조건:

- 소프트웨어는 RV32E ABI 또는 수작업 assembly 수준에서 x0-x15만 사용해야 한다.
- x16-x31을 쓰는 바이너리는 동작 보장 대상이 아니다.

### 2. Data bus / load-store 외부 메모리 기능 포기

적용:

```verilog
parameter integer DISABLE_DBUS = 1
parameter [0:0] WITH_MEM = 0
```

변경 내용:

- `my_mcu_top`에서 data bus request를 `offchip_mem_bridge`로 보내지 않는다.
- `wb_dbus_rdt`는 `32'b0`으로 고정된다.
- `wb_dbus_ack`는 `wb_dbus_cyc`에 즉시 응답하도록 단순화된다.
- `serv_top` 내부에도 `WITH_MEM` 파라미터를 추가해 load/store 관련 decode, memory readback enable, memory interface 일부를 상수화했다.

포기한 것:

- 외부 data memory read.
- 외부 data memory write.
- `LW/LH/LB/LHU/LBU`의 의미 있는 동작.
- `SW/SH/SB`의 의미 있는 외부 저장 동작.
- C 프로그램의 stack, global/static variable, heap 사용.
- 일반적인 메모리 기반 firmware execution model.

남는 조건:

- instruction fetch는 여전히 off-chip memory bridge로 가능하다.
- data side effect가 필요한 프로그램은 사용할 수 없다.
- firmware는 register-only 계산, 즉시값, PC-relative 흐름, 외부 관찰 가능한 RF/제어 상태 위주로 제한해야 한다.

주의:

- 기본 canonical simulation에서는 `DISABLE_DBUS=0`이므로 기존 RV32I directed test는 계속 PASS한다.
- 하지만 physical profile에서는 data bus가 꺼져 있으므로 기존 full RV32I directed regression이 physical profile의 기능 검증을 의미하지 않는다.

### 3. RF write/read interlock 포기

적용:

```verilog
parameter integer USE_RF_INTERLOCK = 0
```

변경 내용:

- instruction fetch ack를 RF write drain 완료까지 지연시키는 보호 로직을 제거했다.
- pending instruction buffer 및 predecode 기반 RF read hint 경로가 비활성화된다.

포기한 것:

- 일반 프로그램에서의 RAW hazard 보호.
- 이전 instruction의 RF write가 외부 serial RF에 완전히 반영되기 전에 다음 instruction이 같은 register를 읽는 경우의 정확성.
- compiler가 자동 생성한 임의 instruction stream에 대한 안전성.

남는 조건:

- firmware 또는 assembly가 RF writeback latency를 고려해 hazard-free하게 작성되어야 한다.
- 필요한 경우 `NOP` 또는 독립 instruction을 삽입해야 한다.
- 이 profile은 일반 CPU가 아니라 timing contract가 있는 firmware-controlled core로 봐야 한다.

### 4. RF 두 번째 read operand 포기

적용:

```verilog
parameter integer SINGLE_RF_READ = 1
parameter single_read_port = 1
```

변경 내용:

- `serv_rf_ram_if`에서 `read_buf1[31:0]` 경로를 사실상 제거했다.
- RF prefetch issue가 32 chunk에서 16 chunk로 줄었다.
- `o_rdata1`은 `1'b0`으로 고정된다.

포기한 것:

- rs2 operand가 필요한 instruction의 정상 동작.
- 대표적으로 다음 instruction class는 동작 보장 대상에서 빠진다.
  - R-type ALU: `ADD/SUB/SLL/SLT/SLTU/XOR/OR/AND/SRL/SRA`
  - 일반 branch 비교: `BEQ/BNE/BLT/BGE/BLTU/BGEU`
  - store data operand가 필요한 store 계열
  - rs2를 의미 있게 읽는 모든 custom/확장 instruction

남는 가능 영역:

- rs1 + immediate 기반 instruction.
- `LUI`, 일부 `AUIPC/JAL/JALR`류 제어 흐름.
- rs2가 0으로 간주되어도 의도한 대로 동작하는 극도로 제한된 code pattern.

중요:

- 이 항목이 마지막으로 1mm 조건을 맞춘 가장 큰 기능 절단이다.
- `SINGLE_RF_READ=0` 상태에서는 placement core가 약 `0.957mm x 0.972mm`까지 내려오기 전 단계에서 `1.017mm x 0.972mm` 수준으로 x축이 1mm를 넘었다.

### 5. 내부 slow clock divider 포기

적용:

```verilog
parameter integer USE_EXT_SYS_CLK = 1
```

변경 내용:

- `i_clk_fast`에서 내부 divider로 `clk_sys`를 만들던 경로를 제거한다.
- 외부에서 별도 slow SERV clock인 `i_clk_sys_ext`를 직접 공급해야 한다.

포기한 것:

- 단일 fast clock 입력만으로 MCU 내부 slow clock까지 생성하는 독립 동작.
- 내부 divider 기반 clocking self-containment.

남는 조건:

- board 또는 Pico 쪽에서 `i_clk_fast`와 `i_clk_sys_ext`를 모두 제공해야 한다.
- 두 clock의 관계, CDC, setup/hold 여유는 system-level에서 관리해야 한다.

주의:

- fast serial clock과 slow SERV clock을 같은 속도로 묶는 방식은 이전 검토에서 실패했다.
- RF/memory serial frame은 여러 fast cycle이 필요하므로 slow core clock은 별도로 느려야 한다.

### 6. reset flop 안정성 일부 포기

적용:

```verilog
parameter RESET_STRATEGY = "NONE"
```

변경 내용:

- SERV 및 bridge 일부 register에서 reset mux 또는 resettable flop 사용을 줄인다.
- 초기값 또는 외부 boot protocol에 더 의존한다.

포기한 것:

- 모든 내부 상태가 reset 입력으로 명시적으로 정리되는 보장.
- simulation/debug 친화적인 deterministic reset behavior 일부.

남는 조건:

- power-up initial value가 합성/공정/flow에서 의도대로 유지되는지 별도 확인이 필요하다.
- tape-out 전 gate-level 또는 post-layout 초기 상태 검증이 필요하다.

### 7. 외부 memory address 폭 포기

적용:

```verilog
parameter integer MEM_ADDR_BITS = 11
```

변경 내용:

- off-chip memory serial frame에서 address field를 32-bit에서 11-bit로 축소했다.
- instruction memory 접근 범위가 줄어든다.

포기한 것:

- 32-bit full address space.
- 큰 firmware image 또는 넓은 memory map.
- 일반적인 memory-mapped address 공간.

남는 조건:

- 접근 가능한 instruction address 범위는 11-bit address 계약에 맞춰야 한다.
- Pico/host 쪽 memory parser도 동일한 축소 protocol을 이해해야 한다.

### 8. conservative placement 여유 포기

적용:

```tcl
set initial_density = 1.00
```

변경 내용:

- QFlow placement density를 0.94에서 1.00으로 올렸다.
- fill 여유와 배선 여유를 줄여 core x/y를 1mm 안쪽으로 눌렀다.

포기한 것:

- conservative placement margin.
- DRC 및 routing congestion에 대한 여유.
- 후속 ECO 여유.

현재 확인된 것:

- `qflow route` 기준으로는 `Final: No failed routes!`.

아직 확인하지 않은 것:

- Magic DRC clean 여부.
- LVS 재검증.
- density 1.00 상태에서의 최종 GDS DRC 안정성.

### 9. 일반 firmware/toolchain 호환성 포기

위 기능 절단들을 합치면 physical profile은 일반 RISC-V compiler output을 직접 실행하는 MCU가 아니다.

포기한 것:

- RV32I GCC 기본 output 실행.
- RV32E라도 일반 ABI/stack/data memory를 사용하는 C firmware 실행.
- load/store, rs2, RAW hazard를 자유롭게 사용하는 code.

남는 조건:

- 전용 hand-written assembly 또는 매우 제한된 compiler backend가 필요하다.
- instruction subset, register allocation, hazard spacing, memory 사용 금지를 모두 firmware contract로 명시해야 한다.

## 면적 감소 단계별 요약

대략적인 흐름은 다음과 같다.

| 단계 | 주요 변경 | Yosys liberty area 또는 physical 결과 |
|---|---|---:|
| 초기 route-clean 계열 | 기능 보존 RV32I off-chip RF/mem | 약 `816k` liberty area, physical 약 `1.7mm²` 이상 |
| 외부 slow clock + reset 축소 | divider 제거, reset NONE | 약 `751k` |
| RV32E | GPR 32개 -> 16개 | 약 `741k` |
| MEM_ADDR_BITS=11 | memory frame address 축소 | 약 `718k` |
| USE_RF_INTERLOCK=0 | RF hazard 보호 제거 | 약 `658k` |
| DISABLE_DBUS=1 | data bus 외부 bridge 차단 | 약 `621k` |
| WITH_MEM=0 | SERV 내부 load/store 제어 상수화 | 약 `579k` |
| ABC area script 개선 | `map -a` 계열 | 약 `562k` |
| SINGLE_RF_READ=1 | rs2 read buffer 제거 | 약 `501k` |

## 현재 physical profile의 성격

현재 결과물은 다음 중 어디에 가까운가?

- 일반 RV32I MCU: 아니다.
- 일반 RV32E MCU: 아니다.
- load/store 없는 RV32E-lite: 부분적으로 맞다.
- rs1/immediate 중심의 proof-of-silicon controller: 가장 정확하다.

즉, MPW에서 RISC-V decode/control, instruction fetch, serial RF/serial memory path 일부를 실리콘으로 검증하기 위한 최소 코어로는 의미가 있다. 하지만 독립 MCU로서 C firmware를 올려 동작시키는 목표와는 거리가 있다.

## tape-out 전에 반드시 정해야 할 것

1. 이 제한 profile을 tape-out 후보로 인정할지 결정해야 한다.
2. 실행할 firmware subset을 문서화해야 한다.
3. `SINGLE_RF_READ=1`, `DISABLE_DBUS=1`, `USE_RF_INTERLOCK=0` 조건을 반영한 전용 testbench를 만들어야 한다.
4. DRC는 아직 보류 상태이므로, 현재 1mm 배치에서 Magic DRC를 다시 확인해야 한다.
5. LVS도 route/migrate 후 재확인해야 한다.
6. reset strategy `"NONE"`의 power-up behavior를 gate/post-layout 수준에서 검증해야 한다.
7. Pico/host 쪽 protocol도 `MEM_ADDR_BITS=11`, data bus 없음, restricted RF access에 맞게 고정해야 한다.

## 결론

면적 조건을 만족시키기 위해 포기한 핵심은 다음 네 가지다.

1. RV32I full register file.
2. data memory load/store.
3. RF RAW hazard 자동 보호.
4. rs2 operand read.

이 네 가지를 포기했기 때문에 1mm x 1mm 안으로 들어왔다.

따라서 현재 physical 결과는 `1mm x 1mm 안에 들어가는 일반 RV32E MCU`가 아니라, `1mm x 1mm 안에 들어가는 제한형 RISC-V-like proof-of-silicon core`로 판단해야 한다.
