## 1. 최종 칩의 탑 모듈 (Top Module)

**네, 맞습니다. 최종 칩의 최상위 탑 모듈은 [my_mcu_top.v](file:///home/hosung/Ho/MPW/MPW_workspace/Bit_Up/A_MCU/src/rtl/my_mcu_top.v)입니다.**

기존의 `serv_rf_top.v`는 내부 RF RAM 중심의 래퍼(Wrapper)로 구성되어 있어 본 설계의 외부 4핀 시리얼 RF 구조를 온전히 반영하기 어렵기 때문에 사용하지 않습니다. 대신 `my_mcu_top.v`에서 다음과 같이 코어와 직렬 통신 어댑터들을 직접 구조적으로 통합하여 최종 칩을 구성하고 있습니다.

```text
my_mcu_top (SoC 최상위 탑 모듈)
  ├─ serv_top (SERV CPU Core)
  │    └─ RF/IBUS/DBUS 인터페이스 포트
  ├─ serv_rf_ram_if (RF 어댑터)
  │    └─ SERV의 bit-serial RF 포트 <=> 2-bit RAM 형식 입출력 변환
  ├─ serv_rf_ram (External RF Serial Bridge)
  │    └─ 2-bit RF 입출력 <=> 4-Pin 외부 RF 시리얼 프레임 변환
  └─ offchip_mem_bridge (External Memory Serial Bridge)
       └─ Wishbone (IBUS/DBUS) <=> 4-Pin 외부 메모리 시리얼 프레임 변환
```

## 2. 최종 칩의 핀 할당 (Pin Allocation)

SOP-28 패키지와 같이 극도로 제한된 핀 리소스를 고려하여 핀 할당은 다음과 같이 최적화되어 선언되어 있습니다.

| 핀 구분 | 포트 이름 | 방향 | 설명 |
| :--- | :--- | :---: | :--- |
| **클럭 및 리셋** | `i_clk_fast` | Input | 외부 고속/기준 클럭 (예: 16MHz) |
| | `i_rst_n` | Input | Active-Low 리셋 신호 |
| **External RF 브리지** | `o_rf_sync` | Output | RF 전송 활성화 동기 신호 (Active-High) |
| *(Pico RF용 4-Pin)* | `o_rf_sck` | Output | RF 직렬 SCK 클럭 출력 |
| | `o_rf_mosi` | Output | MCU -> 외부 RF 명령/주소/데이터 송신 |
| | `i_rf_miso` | Input | 외부 RF -> MCU 데이터 수신 |
| **External Mem 브리지**| `o_mem_sync`| Output | 메모리 전송 활성화 동기 신호 (Active-High) |
| *(Pico Memory용 4-Pin)*| `o_mem_sck` | Output | 메모리 직렬 SCK 클럭 출력 |
| | `o_mem_mosi`| Output | MCU -> 외부 메모리 명령/주소/데이터 송신 |
| | `i_mem_miso`| Input | 외부 메모리 -> MCU 데이터 수신 |
| **UART (2-Pin)** | `o_uart_tx` | Output | UART Transmit 송신 핀 |
| | `i_uart_rx` | Input | UART Receive 수신 핀 |
| **GPIO (24-Pin)** | `i_gpio [7:0]`| Input | 8비트 범용 입력 핀 |
| | `o_gpio [7:0]`| Output | 8비트 범용 출력 핀 |
| | `o_gpio_oe [7:0]`| Output | 8비트 범용 출력 활성화(Output Enable) 제어 핀 |

## 3. 오프칩 직렬 통신 규약 (Communication Protocols)

외부의 Pico 등 에뮬레이터 장치와 데이터를 주고받기 위한 시리얼 전송 규약(프레임 형식 및 타이밍)은 다음과 같이 정의되어 있습니다.

### A. RF 브리지 통신 규약 (`serv_rf_ram.v`)

#### 1) 프레임 포맷 (총 14비트, MSB First 전송)
동기 신호(`o_rf_sync`)가 High 상태인 동안, 14비트의 데이터 프레임이 `o_rf_mosi`를 통해 매 클럭 1비트씩 전송됩니다.
* **프레임 구성**: `req_key = {i_wen, i_ren, target_reg, target_bit, i_wdata}`
  * `[13] WEN` (1비트): 쓰기 활성화 제어 신호
  * `[12] REN` (1비트): 읽기 활성화 제어 신호
  * `[11:6] REG index` (6비트): CPU 레지스터 번호 (CSR 레지스터 포함 0 ~ 35)
  * `[5:2] BIT index` (4비트): 32비트 레지스터 중 접근하는 2비트 청크의 인덱스 (0 ~ 15)
  * `[1:0] WDATA` (2비트): 쓰기 수행 시 전달할 2비트 데이터

#### 2) 읽기(Read) 응답 프로토콜
* 읽기 작업(`REN=1`) 수행 시, 외부 장치(Pico)는 **14비트 송수신 프레임의 가장 마지막 2클럭 동안** `i_rf_miso` 라인에 2비트 값을 순차적으로 실어 보냅니다.
* MCU는 프레임의 마지막 2클럭(`tx_state`가 2와 1인 타이밍)에 `i_rf_miso` 값을 샘플링하여 2비트 데이터를 캡처합니다.

---

### B. 메모리 브리지 통신 규약 (`offchip_mem_bridge.v`)

#### 1) 프레임 포맷 (총 70비트, MSB First 전송)
동기 신호(`o_mem_sync`)가 High 상태인 동안, 70비트의 정보가 `o_mem_mosi`를 통해 전송됩니다.
* **프레임 구성**: `{WE(1), IBUS(1), byte_sel(4), byte_address(32), write_data_or_dummy(32)}`
  * `[69] WE` (1비트): 메모리 쓰기(Write Enable) 활성화 신호
  * `[68] IBUS` (1비트): 명령어 버스(Instruction Bus) 접근인 경우 1, 데이터 버스(Data Bus) 접근인 경우 0
  * `[67:64] byte select` (4비트): 유효한 바이트 레인을 표시하는 Wishbone Sel 값
  * `[63:32] byte address` (32비트): CPU가 접근하려는 32비트 물리 바이트 주소
  * `[31:0] write data` (32비트): 쓰기 수행 시 전달할 32비트 데이터 (읽기 시에는 Dummy 데이터 송신)

#### 2) 읽기/쓰기 응답 프로토콜
* **읽기(Read)**: `WE=0`인 상태에서 프레임의 마지막 32비트(즉, MCU가 Dummy 값을 내보내는 구간) 동안 외부 메모리가 `i_mem_miso`를 통해 32비트 메모리 데이터를 전송하고, MCU는 이를 쉬프트하여 받아 적습니다. 70비트 프레임 전송이 모두 종료되면 CPU에 Wishbone ACK와 함께 데이터를 돌려줍니다.
* **쓰기(Write)**: 70비트 프레임의 송출이 끝나는 즉시 외부 응답 대기 없이 Wishbone ACK를 CPU로 피드백하고 트랜잭션을 마칩니다.

### C. 직렬 클럭 SCK 위상 규약 (공통)
외부 디바이스(Pico)와 데이터 샘플링 타이밍 동기화를 안정적으로 유지하기 위해, SCK 클럭은 다음 규약을 따릅니다.
* 동기화 신호(`sync`)가 High인 동안에만 활성화됩니다.
* 외부 장치인 Pico 보드가 안정적인 셋업/홀드 마진을 갖고 데이터를 캡처할 수 있도록, SCK 클럭 출력은 **고속 클럭의 반전 위상(`~i_clk_fast`)**으로 동작하게 설계되어 있습니다.
