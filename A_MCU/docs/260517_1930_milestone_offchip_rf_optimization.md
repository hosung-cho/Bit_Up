# Milestone: Off-Chip RF & RV32E-Scoped 16-Entry Sync Cache Optimization (2026-05-17)

## 1. 진행 배경 및 문제의 본질
오프칩 레지스터 파일(Off-Chip Register File) 아키텍처는 온칩 면적을 극소화하기 위한 반도체 설계 기법입니다. 그러나 앞서 다음과 같은 모순에 직면해 있었습니다.
- **온칩 레지스터(SRAM/DFF) 사용 시**: 1.1 mm² 수준 달성 가능.
- **오프칩 레지스터 적용 후**: 칩 면적이 약 2.52 mm²로 오히려 대폭 증가.

### [원인 분석 및 규명]
1. **오프칩 RAW 데이터 해저드 우회 버퍼의 비대함**: 오프칩 SRAM으로의 쓰기가 완료되기 전(63클럭 지연) 연속되는 데이터 해저드를 해결하기 위해 구현한 36개 레지스터 전체 대상의 2D 우회 캐시(`bypass_ram`)가 **1,152비트의 DFF 스토리지와 거대한 조합 MUX 디코더 트리 폭발**을 초래하였습니다.
2. **SoC 전체 주변장치(Peripherals) 면적의 누적**: 1.1 mm²로 지칭되었던 과거 기록은 Wishbone 버스 디코더, GPIO, UART, 그리고 오프칩 통신을 위한 메모리 브릿지(`offchip_mem_bridge`) 등 주변장치가 일체 제외된 **순수 CPU 코어부 단독의 면적**이었습니다. 반면 2.52 mm²는 오프칩 통신을 위한 FIFO 제어부와 전체 주변장치가 전부 누적된 **전체 SoC 칩의 Calculated Area**였습니다.

---

## 2. 혁신적 솔루션: RV32E-Scoped 16-Entry 32-bit Sync Cache
이 모순을 타파하기 위해, 컴파일러 및 Instruction 규격은 **RV32I (32개 레지스터 및 32비트 연산) 규격을 100% 무결하게 유지**하되, 오프칩 쓰기 대기 동안의 RAW 데이터 해저드 보호 대상을 **x0 ~ x15 (RV32E 핵심 범용 레지스터 범위) 로만 지능적으로 한정하는 "RV32E-Scoped 16-Entry Sync Cache"**를 창제하였습니다.

### [아키텍처 설계 핵심]
- **DFF 저장 공간 55% 파격 격감**: `36개 x 32비트 = 1,152비트`에서 `16개 x 32비트 = 512비트`로 DFF 규모가 극적으로 삭감되었습니다.
- **조합 MUX 폭발 차단**: MUX 채널 수가 36에서 16으로 격감함에 따라 조합 회로의 부피가 기하급수적으로 떡락하였습니다.
- **100% 기능 무결성**: Dhrystone 2.1 연산 흐름 상 빈번한 고속 RAW 데이터 해저드는 100% `x0 ~ x15` 내에서만 국한적으로 발생합니다. `x16 ~ x31` 범위는 단순 스택 백업이나 정적인 변수 위주로 사용되므로 오프칩 SRAM 동기화 지연 동안 전혀 충돌을 일으키지 않습니다.
