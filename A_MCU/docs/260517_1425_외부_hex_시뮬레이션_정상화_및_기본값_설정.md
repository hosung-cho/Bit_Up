# 외부 Hex 시뮬레이션 정상화 및 기본값 설정

작성 시점: 2026-05-17 14:25 KST

## 1. 개요
6번 외부 hex 기반 테스트벤치(`6_testbench_rv32i_hex`)가 Vivado XSim 환경에서 정상적으로 동작하는지 검증하고, 시뮬레이션 구동 시 매번 환경 변수를 길게 입력해야 하는 번거로움을 해결하기 위한 기본값 설정을 마쳤습니다.

## 2. 현상 분석 및 검증 결과
* **기존 현상**: 이전 문서에서는 외부 hex를 Vivado/XSim simulation에 연결할 때 `unexpected exception when evaluating tcl command` 예외로 인해 실행이 중단된다는 점이 지적되었습니다.
* **재검증 결과**:
  * `sim/build_external_smoke.sh`를 구동하여 `external_smoke.S`를 정상 컴파일하고 `program.hex` 및 기대값들을 빌드하였습니다.
  * `PROGRAM_HEX`, `EXPECTED_HEX`, `EXPECTED_VALID_HEX` 경로 환경 변수를 명시적으로 인가하고 `./run.sh`를 실행한 결과, **아무런 Tcl 에러가 발생하지 않고 시뮬레이션이 성공적으로 완료**되는 것을 확인했습니다.
  * 테스트벤치가 `use_external.flag`를 안정적으로 감지하여 외부 hex 파일 로딩에 돌입했으며, 결과 마커인 `memory[0x700] = 00000001` 비교를 성공적으로 마친 후 `[TB PASS] rv32i directed tests passed on my_mcu_top off-chip path`를 출력했습니다.

## 3. 조치 및 개선 사항
매번 길게 수동으로 환경 변수를 넘겨서 돌려야 하는 단점을 해결하기 위해 `sim/run.sh` 스크립트를 개선했습니다.
* **RTL 코어 보존 원칙**: SERV Core RTL은 전혀 수정하지 않았습니다.
* **개선 대상**: [run.sh](file:///home/hosung/Ho/MPW/MPW_workspace/Bit_Up/A_MCU/testbench/6_testbench_rv32i_hex/sim/run.sh)
* **내용**: 스크립트 실행 시 환경 변수 값이 명시되지 않은 경우, 기본적으로 `generated` 디렉토리에 빌드되어 있는 `program.hex` 및 기대값 파일들의 경로를 가리키도록 디폴트 셸 변수를 선언하여 내보냅니다(`export`).

```bash
# run.sh 수정 사항
export PROGRAM_HEX="${PROGRAM_HEX:-$SCRIPT_DIR/generated/program.hex}"
export EXPECTED_HEX="${EXPECTED_HEX:-$SCRIPT_DIR/generated/expected_mem.hex}"
export EXPECTED_VALID_HEX="${EXPECTED_VALID_HEX:-$SCRIPT_DIR/generated/expected_valid.hex}"
```

이 조치를 통해 사용자는 이제 `sim/` 디렉토리 내에서 단독으로 `./run.sh`만 수행해도 자동으로 외부 hex 빌드 바이너리로 시뮬레이션을 안전하게 수행할 수 있습니다.

## 4. 결론
* 사용자의 지적대로 외부 hex 기반의 빌드 및 시뮬레이션 연동 구조는 아무런 에러 없이 완벽하게 작동하고 있음이 확인되었습니다.
* 외부 hex regression을 위한 시뮬레이션 자동화 인프라의 안정성을 대폭 확보했습니다.
