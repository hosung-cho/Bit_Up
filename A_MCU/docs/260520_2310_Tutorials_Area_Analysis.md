본 보고서는 ETRI 0.5um CMOS MPW 공정 조건 하에서 제공되는 튜토리얼 프로젝트들의 논리 합성 및 Standard Cell Placement 결과를 기반으로 물리적 구현 면적을 비교 분석한 자료입니다.

---

## 1. 개요 및 분석 목적
현재 진행 중인 **SERV MCU** 프로젝트의 1mm x 1mm 다이(Die) 제약 사항을 달성하는 데 있어, 기존 튜토리얼 디자인들의 면적 소모량 및 셀 밀도를 분석하여 우리 디자인의 위치를 객관적으로 파악하고 아키텍처적 개선 방향을 도출하고자 합니다.

## 3. 분석 및 시사점

### ① 1mm x 1mm (1.0 mm²) 면적 제약의 한계선
* **합성 셀 개수 약 1,000개 내외**가 ETRI 0.5um 공정에서 1.0 mm² 안에 패드 구조를 포함하여 안전하게 들어갈 수 있는 마지노선입니다.
* **FIR PE**(884 cells)와 **Pong Game**(1,057 cells)은 Core 면적이 각각 0.49 mm², 0.69 mm²로 1mm² 이내 배치가 가능합니다.
* 반면, **6502 CPU** (2,070 cells)는 density가 0.45로 다소 낮게 설정되어 있으나, 이를 극단적으로 0.9 이상 올리더라도 물리적으로 1.2~1.5 mm² 이하로 줄이는 것이 불가능하여 1mm² MPW 스펙 아웃입니다.
* **FIR8 Filter** (6,857 cells)는 연산기 및 레지스터가 대거 포함되어 있어 아득히 큰 6.13 mm² 면적을 차지합니다.

### ② ETRI 0.5um 공정의 면적 바틀넥: D-FlipFlop (DFF)
* ETRI 0.5um PDK의 Standard Cell 라이브러리에서 순차 회로를 구성하는 DFF는 물리 면적이 매우 큽니다.
  * `DFFPOSX1` (기본 DFF): $36\mu\text{m} \times 24\mu\text{m} = 864 \mu\text{m}^2$
  * `DFFSR` (Set/Reset 포함 DFF): $36\mu\text{m} \times 44\mu\text{m} = 1,584 \mu\text{m}^2$
* 예를 들어, **100개의 기본 DFF**만 배치하더라도 배선 마진을 제외한 순수 DFF 면적만 **0.0864 mm²**를 소모합니다. 
* 따라서 온칩(On-chip) 메모리나 대규모 레지스터 파일(RF)을 Flip-Flop 어레이로 구현할 경우 1mm² 공간은 순식간에 포화됩니다.

## 4. SERV MCU 프로젝트에의 적용 의견

우리가 구현 중인 `my_mcu_top`은 셀 수가 **2,241개** 수준이며, `initial_density = 0.94` 설정 시 Core 면적이 **약 1.71 mm²**에 달합니다. 
6502 CPU 분석 자료(2,070 cells $\rightarrow$ 2.89 mm²)와 비교하면 우리의 MCU 밀도(0.94)가 매우 치밀하게 잘 쪼여진 상태이지만, 절대적인 셀 개수가 2,000개를 넘기 때문에 여전히 1mm x 1mm(1.0 mm²) 물리 다이 규격을 맞추기 힘든 상황입니다.

### 권장 최적화 방향:
1. **레지스터 파일(RF) Off-chip 격리 유지**: 이미 레지스터 파일을 칩 외부에 격리한 구조적 설계는 0.5um 공정에서 면적 제약을 극복하기 위해 필수적인 선택이었습니다.
2. **논리 게이트 축소**: CPU의 제어 및 데이터패스 논리 중에서 불필요한 상태(State) 레지스터나 중복 연산기를 최대한 통폐합하여 합성 후 셀 개수를 최소 **1,200개 이하**로 떨어뜨려야 안정적인 1mm x 1mm 테이프아웃(Tape-out)이 가능합니다.
